Automatic Feature Engineering for Amazon Sales

Project Goal:

Develop an automated system that extracts features from Amazon sales data to improve time series classification model performance.

My Role:

Feature Engineering

Approach:

  • Data Preprocessing:
    • Data Cleaning: Handle missing values, outliers, and inconsistencies in the sales data.
    • Data Transformation: Normalize or standardize numerical features to ensure comparability.
    • Time Series Decomposition: Break down the time series into trend, seasonality, and residual components.
  • Feature Engineering:
    • Statistical Features: Calculate mean, median, standard deviation, variance, skewness, kurtosis, and other statistical metrics.
    • Time-Based Features: Extract features related to time, such as day of week, month, year, and time of day.
    • Trend and Seasonality Features: Capture trends and seasonal patterns using techniques like moving averages, differencing, and Fourier transforms.
    • Domain-Specific Features: Incorporate knowledge of the Amazon sales domain to create relevant features, such as product category, price, and customer reviews.
  • Feature Selection:
    • Employ methods like correlation analysis, feature importance, and recursive feature elimination to select the most informative features.
  • Automated Feature Engineering:
    • Feature Generation: Use automated techniques like genetic algorithms, random forests, and neural networks to discover new features.
    • Feature Selection: Automatically select the best combination of features using techniques like greedy search, hill climbing, and simulated annealing.
    • Hyperparameter Optimization: Tune the parameters of the feature engineering process to maximize model performance.

Tools Used:

  • Python
  • Pandas
  • NumPy
  • Scikit-learn
  • Statsmodels
  • TensorFlow or PyTorch
  • XGBoost
  • Amazon SageMaker
